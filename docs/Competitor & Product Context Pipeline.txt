# Competitor & Product Context Pipeline

**Technical Design Specification**

---

## 1. Purpose and Scope

This document specifies the **downstream context derivation pipeline** used to power:

* **Competitor Tracker**
* **Product Tracker**
* Expanded competitor and product views defined in PRD Section 7 

This pipeline operates **only on finalized canonical Insights** generated by the **Main Insight Orchestrator** defined in the *General Insights Agent Pipeline* .

This pipeline:

* **Does not create insights**
* **Does not modify insight meaning**
* **Does not assign or alter severity, velocity, or category**

Its sole function is to **derive entity-specific context and framing** for presentation.

---

## 2. Design Goals (Derived from PRD)

The pipeline must:

1. Preserve **single-source-of-truth insights**
2. Maintain **consistency across dashboard, competitor, and product views**
3. Support **entity-specific interpretation without reinterpretation**
4. Preserve **explainability and traceability**
5. Avoid **post-hoc judgment or verdicts**

---

## 3. Authoritative Inputs

### 3.1 Canonical Insight Objects

Input insights must already contain:

```
- insight_id
- scope (competitor / product / market)
- time window
- severity
- velocity
- explanation
- category (optional)
```

These insights are **immutable** within this pipeline.

---

### 3.2 Focal Context

Each pipeline execution is parameterized by a **single focal entity**:

```
focal_entity_type: competitor | product
focal_entity_id
```

The focal entity determines filtering logic and lens activation.

---

## 4. High-Level Pipeline Overview

```
Canonical Insights
      ↓
Context Selection Layer
      ↓
Contextual Framing Orchestrator
      ↓
View-Specific Lenses (advisory)
      ↓
Insight Context Blocks
      ↓
UI Presentation
```

---

## 5. Pipeline Components

---

## 5.1 Context Selection Layer (Non-Agent)

### Purpose

Reduce the insight set to only those **relevant to the focal entity** before agent reasoning begins.

### Responsibilities

* Filter insights by:

  * Direct scope match
  * Adjacency relevance (PRD §5.6) 
  * Minimum severity / velocity thresholds
* Enforce tenant and entity scope

### Explicit Non-Responsibilities

* No interpretation
* No ranking
* No explanation generation

This layer exists to prevent **irrelevant insight leakage** into entity views.

---

## 5.2 Contextual Framing Orchestrator (Secondary Orchestrator)

### Role

The **Contextual Framing Orchestrator** is the **single authority** responsible for:

* Ordering insights for a specific entity view
* Grouping insights for presentation
* Coordinating downstream view lenses

This orchestrator **does not synthesize insights** and **does not alter insight attributes**.

---

### Responsibilities

* Determine:

  * Display order
  * Grouping by category (if present)
  * Density and emphasis for the view
* Preserve insight identity and explanation
* Ensure consistent framing rules across views

---

### Explicit Constraints

The orchestrator must never:

* Change severity
* Change velocity
* Change scope
* Assign or override category
* Rewrite the canonical explanation

---

## 5.3 View-Specific Lenses (Advisory Only)

Lenses provide **entity-relative interpretation**, not judgment.

They are **non-authoritative** and **cannot mutate insight objects**.

---

### 5.3.1 Competitor Context Lens

**Activated when**: focal_entity_type = competitor

**Provides**:

* Recurrence indicators
* Escalation vs baseline behavior
* Consistency over time

**Example Outputs**:

```
- recurrence_count
- trend_consistency_note
- historical framing text
```

---

### 5.3.2 Product Exposure Lens

**Activated when**: focal_entity_type = product

**Provides**:

* Explicit adjacency explanation
* Exposure rationale
* Proximity framing

**Must explicitly reference**:

* Adjacency dimension (PRD §5.6.1) 

---

### 5.3.3 Comparative Context Lens (Optional)

**Purpose**:

* Provide peer-relative context

**Provides**:

* Uniqueness vs competitors
* Relative velocity comparison
* Pattern spread indicators

**Constraint**:

* Cannot alter severity or imply recommendation

---

## 5.4 Insight Context Blocks (Primary Output)

### Definition

An **Insight Context Block** is a **presentation-layer construct** that binds:

* A canonical insight
* A focal entity
* View-specific framing

---

### Required Fields

```
- insight_id (reference)
- focal_entity_id
- relevance_summary
- comparative_notes (optional)
- historical_framing (optional)
```

---

### Key Properties

* Multiple context blocks may reference the same insight
* Context blocks are not insights
* Context blocks are safe to cache and reorder

---

## 6. Presentation Mapping (PRD Section 7)

---

### 6.1 Competitor Tracker — List View

Derived from:

* Count of context blocks per competitor
* Presence of high-severity or high-velocity insights

Supports:

> “Which competitors have changed, and where should I look first?”

---

### 6.2 Expanded Competitor View

Uses:

* Context blocks grouped by Theme / GTM / Positioning
* Canonical insight explanations
* Competitor-specific framing overlays

---

### 6.3 Product Tracker — List View

Derived from:

* Context blocks scoped to product
* Direct vs adjacent flags

Supports:

> “Which of our products are affected by recent changes?”

---

### 6.4 Expanded Product View

Uses:

* Product Exposure Lens output
* Explicit adjacency rationale
* Severity and velocity indicators

---

## 7. Invariants (Must Not Be Violated)

1. No insight is created in this pipeline
2. No insight attribute is modified
3. Insight meaning is identical across all views
4. All adjacency must be explicit and explainable
5. Presentation framing must remain non-prescriptive

---

## 8. Summary

This pipeline:

* Preserves the **single-authority insight model**
* Enables rich **competitor and product-specific interpretation**
* Aligns exactly with PRD Section 7
* Prevents semantic drift across UI surfaces
* Cleanly composes with the General Insights Agent Pipeline

---

### Core Principle

> **Insights explain what changed.
> Context explains why that change matters here.**
